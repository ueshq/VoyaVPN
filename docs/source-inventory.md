All reference paths below are relative to the reference root unless an absolute path is shown. The reference repo is read-only evidence; VoyaVPN ports behavior into the target Rust/Tauri/React stack without copying obsolete database columns or legacy migrations.

## Target Module Map

- `crates/voya-core`: OS-free models, enums, parsers, routing/DNS/config generation, config canonicalization, and golden-test helpers.
- `crates/voya-db`: fresh sqlx SQLite schema, repositories, config/default persistence, and model storage boundaries.
- `crates/voya-platform`: OS path resolution, system proxy, PAC support, TUN/elevation, autostart, global hotkeys, tray/platform adapters, and binary permissions.
- `crates/voya-net`: HTTP downloads, subscription fetches, update checks, Clash REST/WebSocket, WebDAV, ruleset/geo fetches, and preset network fetches.
- `crates/voya-udptest`: SOCKS5 UDP-associate channel plus NTP, DNS, STUN, and MCBE testers.
- `crates/voya-app`: app orchestration, typed command handlers, event dispatch, profile/subscription managers, core supervisor, stats manager, and service coordination.
- `src-tauri`: Tauri 2 bootstrap, state injection, generated command/export registration, tray shell, plugins, packaging config, and app lifecycle glue.
- `src`: React app shell, screens/dialogs, Zustand/TanStack Query state, typed IPC wrappers in `src/ipc`, i18next resources, and UI tests.
- `tests/golden`: captured v2rayN-generated Xray/sing-box fixtures and canonical diffs.
