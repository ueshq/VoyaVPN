# Stable External Evidence Checklist

Batch: `04-01-external-evidence-checklist`

This checklist is the release-owner evidence packet for production stable publication. It is completed outside the automated runner by the named human owners. It does not authorize Codex, CI, local scripts, or GitHub Actions to upload artifacts to the CDN, mutate stable pointers, purge caches, use signing secrets, notarize apps, approve legal or privacy gates, or publish a release.

Pointer promotion is blocked until every required entry is complete, the stop or rollback condition has an owner, and the artifact, checksum, pointer, or approval evidence is attached to the release record.

For every row below, complete these per-gate fields in the release record: status, completed-by owner, completion timestamp, evidence links, artifact/hash values, stop or rollback owner, approval record ID when applicable, and residual-risk notes.

## Completion Rule

The release owner may promote stable pointers only when every required checklist entry has an owner, system, required evidence, stop or rollback condition, artifact and hash evidence, and a release-record status. Missing evidence means the release remains staged or blocked; it is not an implicit approval.
